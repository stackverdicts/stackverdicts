  • "5 Signs You Need [Solution]"
  • "How to [Achieve Benefit] Naturally"
  • "The Science Behind [Supplement]"

  • "90-Day Transformation Results"
  • "What I've Learned After Testing 20+ Products"

Why it works: Builds trust and connection, most likely to go viral

Video Script Formula

8-12 Minute Script Structure

HOOK (0:00-0:15) - 15 seconds

Stop the scroll with curiosity, shock, or bold claim

Example: "I spent $197 on TestoPrime so you don't have to. After 30 days, here's what actually happened..."

INTRO (0:15-0:45) - 30 seconds

Who you are, what they'll learn, why they should stay

CONTEXT (0:45-2:00) - 75 seconds

Background, the problem, why this matters

MAIN CONTENT (2:00-9:00) - 7 minutes

  • What is it? (product overview)
  • My experience (week-by-week results)
  • Pros & Cons (be honest!)
  • Who it's for (and who it's NOT for)

CTA (9:00-10:00) - 60 seconds

Link in description, affiliate disclosure, free guide offer

OUTRO (10:00-10:30) - 30 seconds

Like, subscribe, comment, next video teaser

6. System 2: Content Calendar Dashboard

Overview

Visual planning system to organize your video production pipeline from script generation through publishing. Think of it as your YouTube production command center.

Why You Need This

The Consistency Problem

The #1 reason YouTube channels fail is inconsistent posting. Without a system, you'll:

  • Miss publishing deadlines
  • Scramble for content ideas last minute
  • Lose track of which videos are in production
  • Duplicate effort or forget about completed scripts

Solution: Calendar view shows exactly what's scheduled, what needs work, and what's ready to publish.

Core Features

  • Calendar View: Month/week view of all content with color-coded status
  • Status Tracking: Draft → Recorded → Editing → Scheduled → Published
  • Drag & Drop: Move videos to different dates easily
  • Quick Actions: Update status, add notes, link to YouTube video
  • Batch Scheduling: Schedule multiple videos at once
  • Analytics Integration: See performance directly on calendar

Database Schema

CREATE TABLE content_calendar (
  id VARCHAR(36) PRIMARY KEY,
  script_id VARCHAR(36),
  scheduled_date DATE NOT NULL,
  status ENUM('draft', 'recorded', 'editing',
              'scheduled', 'published') DEFAULT 'draft',
  priority ENUM('low', 'medium', 'high') DEFAULT 'medium',
  notes TEXT,
  recording_deadline DATE,
  editing_deadline DATE,
  thumbnail_ready BOOLEAN DEFAULT FALSE,
  youtube_video_id VARCHAR(50),
  actual_publish_date TIMESTAMP,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (script_id) REFERENCES youtube_scripts(id),
  INDEX idx_scheduled_date (scheduled_date),
  INDEX idx_status (status)
);

API Endpoints

Workflow Integration

Complete Production Pipeline

  1. Monday - Script Generation:
    • Generate 3 scripts using System 1
    • Immediately add to calendar for next 3 publishing slots
    • Set status: "Draft"
  2. Tuesday - Recording:
    • Calendar shows which 3 videos need recording
    • After recording, update status: "Recorded"
    • Add notes: "Recorded in one take" or "Need to re-shoot intro"
  3. Wed-Fri - Editing:
    • Calendar shows videos in "Recorded" status
    • Move to "Editing" when you start
    • Check off "thumbnail_ready" when complete
  4. Friday Evening - Scheduling:
    • Upload videos to YouTube
    • Add youtube_video_id to calendar event
    • Update status: "Scheduled"
  5. Auto-Update on Publish:
    • When video goes live, cron job updates status: "Published"
    • Sets actual_publish_date timestamp

Automation Workflow

How It All Works Together

  1. Lead Capture:
    • Viewer lands on landing page from YouTube
    • Enters email to claim offer
    • POST /api/email/subscribers creates new subscriber
    • Tags them with offer interest: ["testosterone", "high-intent"]

  4. Tracking:

9. System 5: YouTube Analytics Integration

Overview

Automated system that syncs video performance data from YouTube API and correlates it with affiliate conversions to calculate true ROI per video.

The Missing Link: Video Performance → Revenue

Most YouTubers know their views, likes, and watch time. But they don't know:

  • Which videos generate the most revenue
  • True ROI per video (revenue ÷ production cost)
  • Which traffic sources convert best
  • Optimal video length for conversions

This system connects YouTube Analytics to MaxBounty conversions so you know exactly which content to create more of.

Core Features

  • Automatic Data Collection: Sync video analytics daily via YouTube Data API
  • Revenue Attribution: Match conversions to source videos
  • Performance Trends: Track view velocity, retention curves, engagement over time
  • Traffic Source Analysis: Know if viewers came from YouTube Search, Suggested, or External
  • Channel Overview: Aggregate stats across all videos
  • Top Performers: Automatically identify highest-earning videos

Statistical Significance Explained

Understanding the Math (Simplified)

Statistical significance tells you: "Are these results real, or just random luck?"

95% Confidence Level = 95% sure the winning variant is actually better (only 5% chance it's a fluke)

How It's Calculated:

  1. Sample Size: Need enough data (minimum 100 conversions per variant recommended)
  2. Conversion Rate Difference: How much better is Variant A vs Control?
  3. Standard Error: Measures variability in the data
  4. Z-Score: Mathematical test comparing the two variants

Rule of Thumb:

  • Green (95%+ confidence) = Safe to declare winner
  • Yellow (80-95%) = Trending, but keep running
  • Red (<80%) = Not enough data yet

async function calculateABTestResults() {
  const runningTests = await query('SELECT * FROM ab_tests WHERE status = ?', ['running']);

  for (const test of runningTests) {
    const variants = await query('SELECT * FROM ab_test_variants WHERE test_id = ?', [test.id]);

    // Aggregate events for each variant
    for (const variant of variants) {
      const stats = await query(`
        SELECT
          SUM(CASE WHEN event_type = 'impression' THEN 1 ELSE 0 END) as impressions,
          SUM(CASE WHEN event_type = 'click' THEN 1 ELSE 0 END) as clicks,
          SUM(CASE WHEN event_type = 'conversion' THEN 1 ELSE 0 END) as conversions
        FROM ab_test_events
        WHERE test_id = ? AND variant_id = ?
      `, [test.id, variant.id]);

      // Calculate conversion rates
      const conversionRate = stats.conversions / stats.impressions;
      const clickRate = stats.clicks / stats.impressions;

      // Update results table
      await query(`
        INSERT INTO ab_test_results (id, test_id, variant_id, impressions, clicks, conversions)
        VALUES (?, ?, ?, ?, ?, ?)
        ON DUPLICATE KEY UPDATE
          impressions = VALUES(impressions),
          clicks = VALUES(clicks),
          conversions = VALUES(conversions),
          calculated_at = NOW()
      `, [generateId('result'), test.id, variant.id, stats.impressions, stats.clicks, stats.conversions]);
    }

    // Calculate statistical significance
    const significance = await calculateStatisticalSignificance(test.id);

    if (significance.confidence >= test.confidence_level) {
      // Winner found!
      await query('UPDATE ab_tests SET winning_variant = ? WHERE id = ?',
        [significance.winningVariant, test.id]);
    }
  }
}

      const script = await contentService.generateScript({
        topic: topic.topic,
        offerIds: topic.offerIds,
        tone: request.tone || 'professional',
        videoLength: request.videoLength || 'medium',
        includeHooks: true
      });

      results.push({
        success: true,
        scriptId: script.id
      });
    } catch (error) {
      results.push({
        success: false,
        error: error.message
      });
    }
  }

  return {
    total: request.topics.length,
    successful: results.filter(r => r.success).length,
    failed: results.filter(r => !r.success).length,
    results
  };
}

// In the service layer
private parseSettingValue(setting: Setting): Setting {
  if (setting.setting_value === null) return setting;

  let parsedValue: any = setting.setting_value;

  switch (setting.setting_type) {
    case 'number':
      parsedValue = parseFloat(setting.setting_value);
      break;

    case 'boolean':
      parsedValue = setting.setting_value === 'true';
      break;

    case 'json':
      try {
        parsedValue = JSON.parse(setting.setting_value);
      } catch {
        // If JSON parsing fails, return as string
        parsedValue = setting.setting_value;
      }
      break;

    case 'string':
    default:
      // Already a string, no parsing needed
      parsedValue = setting.setting_value;
  }

  return { ...setting, setting_value: parsedValue };
}

// Wrapper around job execution
private async executeJob(jobName: string, task: () => Promise): Promise {
  const executionId = generateId('exec');
  const startTime = Date.now();

  try {
    // Get job from database
    const jobRows = await query>(
      'SELECT id FROM cron_jobs WHERE name = ?', [jobName]
    );

    if (jobRows.length === 0) {
      logger.error('Scheduler', `Job not found: ${jobName}`);
      return;
    }

    const jobId = jobRows[0].id;

    // Record execution start
    await query(
      `INSERT INTO cron_job_history (id, job_id, started_at, status)
       VALUES (?, ?, NOW(), 'running')`,
      [executionId, jobId]
    );

    logger.info('Scheduler', `Starting job: ${jobName}`);

    // Execute the actual task
    const result = await task();

    const executionTime = Date.now() - startTime;

    // Record success
    await query(
      `UPDATE cron_job_history
       SET completed_at = NOW(),
           status = 'success',
           execution_time_ms = ?,
           metadata = ?
       WHERE id = ?`,
      [executionTime, JSON.stringify(result || {}), executionId]
    );

    // Update job status
    await query(
      `UPDATE cron_jobs
       SET last_run = NOW(),
           last_status = 'success',
           last_error = NULL,
           run_count = run_count + 1
       WHERE id = ?`,
      [jobId]
    );

    logger.info('Scheduler', `Completed job: ${jobName} in ${executionTime}ms`);

  } catch (error) {
    const executionTime = Date.now() - startTime;

    // Record error
    await query(
      `UPDATE cron_job_history
       SET completed_at = NOW(),
           status = 'error',
           execution_time_ms = ?,
           error_message = ?
       WHERE id = ?`,
      [executionTime, error.message, executionId]
    );

    // Update job with error
    await query(
      `UPDATE cron_jobs
       SET last_run = NOW(),
           last_status = 'error',
           last_error = ?,
           run_count = run_count + 1
       WHERE name = ?`,
      [error.message, jobName]
    );

    logger.error('Scheduler', `Job failed: ${jobName}`, { error: error.message });
  }
}

this.scheduleJob('offer-sync', '0 2 * * *', async () => {
  const stats = {
    updated: 0,
    added: 0,
    removed: 0,
    errors: 0
  };

  try {
    // Fetch offers from MaxBounty
    const apiKey = await settingsService.getSetting('maxbounty_api_key');
    const affiliateId = await settingsService.getSetting('maxbounty_affiliate_id');

    const response = await fetch(`https://api.maxbounty.com/offers?key=${apiKey}&aff_id=${affiliateId}`);
    const offers = await response.json();

    for (const offer of offers) {
      try {
        // Check if offer exists
        const existing = await query(
          'SELECT id FROM offers WHERE external_id = ?',
          [offer.campaign_id]
        );

        if (existing.length > 0) {
          // Update existing offer
          await query(
            `UPDATE offers
             SET name = ?, payout = ?, description = ?, updated_at = NOW()
             WHERE external_id = ?`,
            [offer.campaign_name, offer.payout, offer.description, offer.campaign_id]
          );
          stats.updated++;
        } else {
          // Add new offer
          await query(
            `INSERT INTO offers (id, external_id, name, payout, description, network)
             VALUES (?, ?, ?, ?, ?, 'maxbounty')`,
            [generateId('offer'), offer.campaign_id, offer.campaign_name, offer.payout, offer.description]
          );
          stats.added++;
        }
      } catch (error) {
        stats.errors++;
        logger.error('OfferSync', `Failed to sync offer ${offer.campaign_id}`, { error });
      }
    }

    // Mark offers not in API response as inactive
    const apiOfferIds = offers.map(o => o.campaign_id);
    const removed = await query(
      `UPDATE offers
       SET is_active = FALSE
       WHERE network = 'maxbounty'
       AND is_active = TRUE
       AND external_id NOT IN (?)`,
      [apiOfferIds]
    );
    stats.removed = removed.affectedRows;

    return stats;

  } catch (error) {
    logger.error('OfferSync', 'Failed to sync offers', { error });
    throw error;
  }
});

          to: email.email,
          subject: email.subject.replace('{{first_name}}', email.first_name),
          body: email.body.replace('{{first_name}}', email.first_name)
        });

        // Mark as sent
        await query(
          'UPDATE sequence_emails SET sent_at = NOW() WHERE id = ?',
          [email.sequence_email_id]
        );

        // Advance to next email
        await query(
          `UPDATE subscriber_sequences
           SET current_email_index = current_email_index + 1
           WHERE id = ?`,
          [email.subscriber_sequence_id]
        );

        stats.sent++;

      } catch (error) {
        if (error.code === 'BOUNCE') {
          stats.bounced++;
          // Mark subscriber as bounced
          await query(
            'UPDATE subscribers SET status = ? WHERE id = ?',
            ['bounced', email.subscriber_id]
          );
        } else {
          stats.errors++;
        }
        logger.error('EmailSequence', `Failed to send email to ${email.email}`, { error });
      }
    }

    return stats;

  } catch (error) {
    logger.error('EmailSequence', 'Failed to process email sequences', { error });
    throw error;
  }
});

Pro Tips for Reliable Automation

  • Idempotent Jobs: Job should produce same result if run multiple times (important if job runs twice accidentally)
  • Timeout Protection: Set max execution time. Kill job if exceeds (prevents hung jobs)
  • Graceful Degradation: If external API fails, log error and continue. Don't crash entire job
  • Batch Processing: Process items in batches of 100, not all at once. Reduces memory usage
  • Off-Peak Scheduling: Heavy jobs (offer sync, analytics) run at 2-4 AM when traffic is low
  • Database Locks: Use row-level locks to prevent race conditions if job runs concurrently
  • Retry Logic: For transient failures (network), retry 3 times with exponential backoff before failing
  • Job Dependencies: If Job B depends on Job A, schedule B 30 min after A completes
  • Logging is Critical: Log start/end, stats, errors. You'll debug jobs based on logs
  • Test Locally: Before deploying new job, test manually with "Run Now" button
